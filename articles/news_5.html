💡 AI 深度解析 (AI Insight)

Craft Agent 的出现，深刻反映了AI Agent技术从早期命令行接口（CLI/TUI）向更用户友好图形界面（GUI）演进的趋势。核心概念在于它将AI Agent的交互、技能（Skill）、API及多控制器协议（MCP）等复杂功能，通过『文档中心化管理』的方式呈现。这意味着每一次与Agent的对话或任务执行都被视为一个可编辑、可追溯的文档，用户可以方便地标记状态、打标签、进行富文本渲染和导出，极大地提升了AI Agent的可管理性和可视化程度。这种设计理念对行业产生了显著影响。首先，它降低了AI Agent的使用门槛，让非技术用户也能高效利用AI能力，从而实现AI能力的普及化。其次，通过整合多种LLM（如Claude、ChatGPT、Ollama）和自动化任务（定时、事件驱动、智能体驱动），Craft Agent 提供了一个统一且强大的工作平台，优化了开发者的工作流，并为企业级应用提供了灵活的定制与部署方案。其开源属性（Apache 2.0）进一步鼓励了社区共建和创新。未来，随着此类GUI Agent客户端的普及，AI Agent有望更深度地融入各类桌面及移动生产力工具，从根本上改变人机交互模式，提升各行各业的生产力与创造力。
Original Text
平台：macOS / Windows / Linux

关键词：AI Agent

@Vanilla：「AI Agent 时代来临了」这件事，大家已经达成共识。但一个还没被认真回答的问题是：Agent 应该长什么样？ 是终端（Coding CLI）？是聊天框（Chatbot）？还是工作流画布（Canvas）？Craft Agent 给出了一个截然不同的答案，而且它把答案做成了一个开源的桌面应用。

当我知道 Craft 做了一个 AI Agent 应用时，我的第一反应是这是一款对标 Notion AI Agent 的产品。然而，Craft 团队这次出乎了我的意料。可以说，Craft Agent 这款应用和 Craft 本体毫无关系，而是为 AI Agent 量身定制了一个文档中心化的桌面级 GUI。换句话说，如果你用过 Codex 的桌面端应用，你应该可以理解这款产品的定位，它聚合了 AI Agent、Skill、MCP 等功能，但是以「文档 / 会话」形式存在，形成了可读、可管理、可回溯的特性。

根据 GitHub 上官方仓库的介绍，Craft Agent 是基于 Claude Agent SDK 和 Pi SDK 打造，支持 Claude 订阅、ChatGPT 订阅、GitHub Copilot 订阅、API Key、本地模型（Ollama）等多种使用 LLM 的方式，而且这个应用是基于 Apache 2.0 开源协议的一个开源应用，可以自部署或者二次开发。

看到这里，也许你就明白了，Craft Agent 本质上是目前流行的 Coding CLI 的 第三方 GUI 客户端，面向那些不喜欢在终端 app 里操作 Claude Code、Codex、Gemini CLI、Opencode 等 TUI 产品的用户。那么接下来，我们就来看看 Craft Agent 都有哪些核心功能。

文档中心化管理

在 Craft Agent 中，会话（session）以文档的形式被管理，既可以根据完成状态进行标记，比如说 backlog、todo、needs review、done、cancelled 等，也可以根据内容打上标签（Labels），当然你可以对这些 session 进行旗标、归档等操作。在这种文档中心化的管理方式下，所有 session 都可以非常直观、高效地进行整理、搜索、调用，大多数情况下体验都比终端 app 中的 TUI Agent 要更好。

在每个 session 中，我们可以方便地切换操作模式（包括 explore、ask to edit、execute 三种），也可以直接更改待办状态，而最让我喜欢的是 Craft Agent 对 Markdown 文件的渲染样式。在 session 的输出结果中，Craft Agent 会直接以富文本的形式来展示图文类的输出结果，简洁、优雅的渲染风格非常符合 Craft 团队的产品调性。

当你需要导出 session 时，你可以直接将整个 session 以在线文档的形式通过链接来进行分享，也可以对其中的 Markdown 文件复制富文本或者 Markdown 源代码。

扩展能力管理

在使用 AI Agent 的过程中，不可避免地会用到各种 Skill、API 和 MCP。在 TUI 中，这些配置以本地文件的形式被储存和调用，在 TUI 中需要通过各种指令来进行查看。在 Codex 应用中，我们可以在设置界面中管理这些扩展能力。而在 Craft Agent 中，我们可以直接在侧边栏中就管理、查看、新增 Sources（包括 APIs、MCPs、本地文件夹）和 Skills。当然，这些配置文件也在本地被储存，方便同步、备份。

自动化任务管理

最近大火的 OpenClaw 有一个非常实用的功能是 Cron 任务，可以定时让 AI Agent 自动运行任务并返回结果。在 Craft Agent 中，我们也可以非常方便地创建自动化任务，支持 Scheduled、Event-Based、Agentic 三种触发方式。我在 Craft Agent 中制作了一个图标来简要解释了这三种自动化类型的实现逻辑和区别。

经过一段时间的使用，我认为 Craft Agent 确实是一款非常优秀的 AI Agent 客户端，比 TUI 界面更加适合绝大多数用户，特别适合在主力设备上使用，因为它可以方便、直观地管理会话、扩展能力和自动化任务。如果一定要说一个缺点，那就是 Craft Agent 现在只有桌面端，如果能补全移动端和远程操作的能力，我相信它会有更加广阔的前景。
