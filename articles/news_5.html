派评 | 近期值得关注的 App

💡 AI 深度解析 (AI Insight)

本文介绍了一系列提升数字生活效率与安全的应用，其核心专业概念在于“工具链整合与自动化”、“数据主权与隐私保护”以及“信息智能化聚合”。Chops通过提供一站式AI Agent技能管理，解决了碎片化存储和跨工具检索的痛点，体现了AI时代开发工作流对“互操作性”和“统一管理”的迫切需求。brrr利用Webhooks技术实现跨设备、跨服务的通知统一，展示了“自动化”和“事件驱动通信”如何将分散的系统信息集中化。PaperKnife+和MetaPurge则强调了“数据安全”与“隐私管理”，通过离线操作和EXIF信息清除，赋能用户对个人数据拥有“主权”。Flighty的AirPort Intelligence功能则利用“数据分析”和“实时信息聚合”来预测并解释航班延误，显著提升了出行场景的“决策效率”。这些工具共同描绘了一个趋势：未来的应用不再是孤立存在，而是通过智能化、安全化的方式深度集成，以更高效、更安全、更用户友好的方式服务于个人与专业场景。
Original Text
欢迎收看本期《派评》。你可以通过文章目录快速跳转到你感兴趣的内容。如果发现了其它感兴趣的 App 或者关注的话题，也欢迎在评论区和我们讨论。
@Vanilla：随着 AI 编程工具的普及，Skills 已经成为定制和扩展 Agent 行为的核心机制。但问题在于，每个工具都有自己的本地存储路径，散落各处的 Skills 文件难以形成统一管理，更谈不上跨工具检索。Chops 的出现正是为了解决这一痛点，它支持 Claude Code、Cursor、Codex、Windsurf、Copilot、Aider、Amp 等工具，可以实现一站式 Skills 管理。
首先，Chops 内置了一个编辑器，可以对 Skills 文件直接进行查看、编辑、保存等操作，免去了跳转到第三方文本编辑器的繁琐。除了 Markdown 格式的文件，Chops 还可以自动解析 YAML Frontmatter（元数据）和 Cursor 特有的 .mdc 文件格式。值得一提的是，Chops 还支持新建 Skills，并会自动生成对应工具的规范模板（boilerplate）。
其次，Chops 支持跨工具的整理、搜索和同步功能。Chops 可以自动识别本机上的 Skills，并按照不同的工具分类显示。对这些 AI Agents 中散落的 Skills，我们可以通过 Chops 中的喜爱和合集功能进行整理，帮助我们在需要时快速调用。如果你懒得整理也没有关系，Chops 支持对 Skills 文件的全文搜索，包括名称、描述和内容。Chops 对 Skills 文件的索引是基于 macOS 的 FSEvents 技术，当我在终端或其它编辑器修改了 Skills 文件，Chops 会实时刷新显示。
最后，Chops 还支持对远程服务器中 Skills 的管理。以 OpenClaw 为例，我们可以通过服务器信息和密钥来建立连接，然后让 Chops 远程管理 OpenClaw 中的 Skills，包括搜索、编辑、删除、安装等操作都不在话下。
如果你只想要一个纯粹的 Skills 管理工具，可以说 Chops 的功能已经相当完善，对于日常与多个 AI 编程工具打交道的开发者来说，Chops 是一个值得纳入工作流的效率工具。Chops 目前免费开源，你可以在官网直接下载安装，也可以从 GitHub 平台下载源码自行编译安装，不过目前仅支持 macOS 平台（需要 macOS 15 Sequoia 或更高版本）。
@大大大K：现在手机系统大都支持拍照直接生成 PDF 扫描件，我们也越来越习惯这样操作，而不是再去找扫描仪了。但这种方便往往只停留在第一步，如果扫描之后发现还需要单独调整某个页面，或者要把多个 PDF 合并、拆分时，身边要是没有电脑就很难办了。如果此时我们手机里有这样一个能简单操作 PDF 文件的 App，那就方便很多了。
PaperKnife+ 就是这样一款 PDF 工具。它的 UI 设计非常出色，界面中的各种阴影、图标等细节非常精致，完全没有传统「工具型」App 那样沉重。功能排布采用非常直观的卡片式设计，清晰又简洁。虽然 App 不支持中文显示，但也都是非常简单的英文描述，大概一看就能看懂。体验下来非常轻盈丝滑，质感极佳。
PaperKnife+ 对于 PDF 的编辑功能非常强大，足以应付我们日常使用的各类需求，比如常用的合并、文件压缩、页面旋转等。此外它还有很多高阶功能供我们使用，例如添加水印、签名，甚至连加密、编辑元数据等功能都一应俱全。如果你想将 PDF 转换为其它格式，PaperKnife+ 同样也能满足你，能够直接在 PDF 和图片中互相转换，还可以将其打包成 ZIP 压缩文件，甚至能提取扫描版的 PDF 文字，实测中文识别效果也非常不错。
当然了，作为一款需要处理我们私有文件的 App，数据安全至关重要。PaperKnife+ 本身是开源作品，并且所有处理功能完全都是离线操作，这也是 PaperKnife+ 的初衷，开发者希望用户能够放心、安全地使用这款 App 来操作含有账单、银行卡、身份证信息的文件。那么如果你平时也有用手机操作 PDF 文件的需求，可以在 GitHub 或 IzzyOnDroid 下载 PaperKnife+，安装一个有备无患。
@Snow：通知，是目前电子设备上不可或缺的一项功能，无论是天气预报、日程提醒还是消息通知，我们早已习惯借助通知来了解事物的变化。不过，即使手机可以统筹所有安装应用的通知，生态可以连接不同设备的通知，总有一些通知会被区隔开，也总有一些变化是沉默的，而 brrr 或许可以帮你划开一些区隔，敲响一些沉默。
首次打开 brrr 可能会让人一脸懵，首屏只有一段命令行，点击命令行右上角的 Send Test 按钮，你会在已安装的设备上收到显示为「Hello world!」的推送通知，底部则设置了复制和分享按钮。此时，如果你不小心将命令行复制到终端工具中，或是意外打开了官方文档，就会发现 brrr 的真正妙用。
在功能设计上，brrr 并不会参与任务执行，也不负责触发逻辑，而是专注在「结果传递」这件事上。点击复制按钮，你可以获得一个专属的密钥和推送地址，只要利用 Webhooks 向这个地址发送一段内容，这段内容就会以系统通知的形式出现在你的设备上。brrr 支持自定义通知的主副标题、消息内容、通知声音、通知分组、通知模式等，甚至还能附带跳转链接以及设置过期时间。只要你使用的设备或服务，支持发送 HTTP 请求，它就可以把消息借由 brrr 传递给你。
了解 brrr 的原理，我第一时间想到可以同步我 Synology NAS 中各类下载工具的消息。你只需要进入「DSM 后台-控制面板-通知设置-Webhooks」中新增规则，将 Download Station、qBittorrent 等下载工具的下载成功、下载失败两类情况区分设置规则，然后填入 brrr 提供的推送地址、推送内容等信息，brrr 就会在文件下载成功或失败后，将对应消息推送给你。如果你不知道 JSON 文档该如何撰写，官方文档内置了测试工具，你可以参照每一个空白处的介绍填写，完成测试后一键复制，或是再做二次修改。
与之类似，你还可以将 brrr 与备份任务完成、Docker 服务异常这些 NAS 通知，以及其它自动化流程、脚本或设备结合起来，比如 Home Assistant、旁路由等，作为整套自动化体系最后的收束，你便不再需要主动打开各类应用或后台查看状态，而只在需要的时候被「统一」告知结果。
你可以在 App Store 下载 brrr，应用目前支持 iPhone、iPad 及 Mac 设备，并提供了 2 周免费试用，此后可按 8 元/月 或 78 元/年 的价格订阅使用。考虑跨设备、跨服务通知统筹需求目前比较小众，建议你结合自身情况，试用后再行决定是否付费。
@Peggy_：近年来，社交媒体上不乏一些关于个人隐私的提醒， 被提醒最多的便是分享照片可能带来的隐私泄露问题。事实上，在我们随手分享的日常照片中很可能包含了大量隐私信息，拍摄地点、拍摄日期、设备名称等等，随着 AI 技术的发展，原本需要人工破译的信息都能瞬间被 AI 获取。
那么为了我们的隐私安全，我们可以先从图片的处理做起。常见的处理方式有打码，删除照片 EXIF 信息等。今天要介绍的 MetaPurge 就是一款专注于照片信息删除的小工具。
其使用方式也非常简单，打开应用，在主界面点击添加图片，添加完成后 MetaPurge 能够读取图片所包含的各类信息，所以某种程度上 MetaPurge 也能作为一个 EXIF 信息查看器。确认好要删除信息的图片后，点击 Purge 即可开始删除，比较贴心的是，MetaPurge 并不会直接将我们原图的信息删除，而是允许我们另存一份干净的图片至相册。这样做的好处是当我们出于信息整理的目的需要原图时，不至于因为图片信息丢失而无从下手。
除了单独删除，MetaPurge 还支持批量选择图片来删除相应的信息。实际测试下来，经过 MetaPurge 处理的图片，基本上就只剩下图片大小这一信息了。
值得一提的是，MetaPurge 并不会获取网络权限，所有操作都在本地完成，我们无需担心其会窃取我们的任何隐私。
如果你正在寻找具备类似功能的小工具，不妨通过 F-Droid 下载体验 MetaPurge，应用开源且完全免费。
@化学心情下2： 坐飞机最害怕的事情是什么？一定是航班延误，航班延误往往与天气情况或者机场限流等因素有关，那么了解到起飞的机场信息就非常重要了，Flighty 最新推出的 AirPort Intelligence 功能就可以直观地让你看到航班延误的原因是什么，还可以帮你预测未来延误结束的时间。
在机场信息面板中，你可以查看今天航班的起降情况，包括平均延误时间、实时天气、机场整体的起飞和到达延误率、航班取消情况，以及各个小时的延误时长和天气变化。
页面下方还会继续展示机场内各家航空公司与航班的繁忙程度，以及目的地和中转地排名等信息。以南京为例，当前航班量最多的目的地是广州白云机场和深圳宝安机场，而执飞架次最多的航空公司则是中国东方航空。
点击左下角的「起飞航班」按钮，可以查看当日起飞航班的整体延误情况，以及各航班的具体延误时长；点进单个航班后，还能进一步查看完整行程信息。顶部筛选栏也支持按航司、目的地等条件进行筛选。对应的「到达航班」按钮提供的信息大体相同，这里不再展开。
整体来看，此次升级加入的 Airport Intelligence 功能，进一步提升了 Flighty 在出行场景中的实用性。它可以帮助用户在出发前更全面地了解机场运行情况，提前判断可能出现的延误风险等等。如果你感兴趣，可以在 App Store 下载 Flighty，Airport Intelligence 需要 Pro 计划才可解锁，你也可以通过官网查看 Flighty 提供的真实航班数据示例。
